Introduction

The following vignette aims at showing an example workflow for performing PK Ratios plots and qualifications using the tlf-Library. The very first step of the workflow is to load the tlf library and associated packages (so far ggplot2). The 2 following steps are the definition of the dataMapping and plotConfiguration.

  • dataMapping aims at linking which variable from the data, metaData should be plotted.
  • plotConfiguration aims at defining how this map data should be plotted.
useTheme(tlfTheme)

For this example, the data and its metaData will be called pkRatioData and pkRatioMetaData. pkRatioData is expected to be a data.frame and pkRatioMetaData a list of lists containing, for each variable of pkRatioData, its unit and dimension. Data Mapping

To map which data to plot, DataMapping objects derived from R6Class can be used. For PK Ratio plots, PKRatioDataMapping is the most suited mapping: Default x is "Age" and default y is "Ratio". However, this variables can be overwritten. The class defines pkRatioLines as limits to be plotted in the PK Ratio plot with default values predefined. Since PKRatioDataMapping inherits from an XY Group Mapping (XYGDataMapping), it allows as well the definition of groupings.

Example of Definition of PK Ratio Data Mapping

# Create a PK Ratio dataMapping
groupings <- Groupings$new(color = "Gender", 
                           shape = c("Dose", "Compound"))

dataMapping <- PKRatioDataMapping$new(x = "Age", y = "PKRatios", groupings = groupings)


dataMapping$x
#> [1] "Age"
dataMapping$y
#> [1] "PKRatios"
dataMapping$pkRatioLines
#> [1] 1.0000000 1.5000000 0.6666667 2.0000000 0.5000000
dataMapping$groupings$color$group
#> [1] "Gender"
dataMapping$groupings$shape$group
#> [1] "Dose"     "Compound"

Plot Configuration

To configure the plot properties, PlotConfiguration objects derived from R6Class can be used. They associate the following features:

  • LabelConfiguration, inherited R6Class defining the labels of the plot (title, subtitles, x axis, y axis and watermark)
  • LegendConfiguration, R6Class defining the legend properties of the plot
  • xAxisConfiguration, R6Class defining the properties of the X axis (scale, limits, printed values)
  • yAxisConfiguration, R6Class defining the properties of the Y axis (scale, limits, printed values)

For PK Ratio Plots, PKRatioPlotConfiguration is the most suited of PlotConfiguration The most suited way is to provide data, metaData and dataMapping to get automatically the associated plot configuration. However, any of the properties can be overwritten (either by indicating the updated input or by redifing the value after the creation of the object).

Example of Definition of PK Ratio Plot Configuration:

# Create PK Ratio data Maping
dataMapping <- PKRatioDataMapping$new()

plotConfiguration <- PKRatioPlotConfiguration$new(
  data = pkRatioData,
  metaData = pkRatioMetaData,
  dataMapping = dataMapping
)

Plot PK Ratios

The function plotPKRatio allows for an easy way of plotting PK Ratios with specific PK Ratio limits.

  • Input arguments of plotPKRatio are the data, metaData, their dataMapping and `plotConfiguration
  • Output argument of plotPKRatio is the plotObject corresponding to a ggplot object.

If no dataMapping and plotConfiguration is provided, plotPKRatio uses a default mapping and configuration.

Example of PK Ratio Plot workflow

# Create PK Ratio data Mapping
pkRatioDataMapping <- PKRatioDataMapping$new(
  groupings = Groupings$new(color = "Gender",
                            shape = c("Dose", "Compound")))

# Create associated PK Ratio plot Configuration
pkRatioPlotConfiguration <- PKRatioPlotConfiguration$new(
  ylabel = "Simulated/Observed Cmax Ratio",
  data = pkRatioData,
  metaData = pkRatioMetaData,
  dataMapping = pkRatioDataMapping
)

# Define the printable ggplot object
plotObject <- plotPKRatio(
  data = pkRatioData, 
  metaData = pkRatioMetaData,
  dataMapping = pkRatioDataMapping, 
  plotConfiguration = pkRatioPlotConfiguration
)

plotObject

Qualification of PK Ratios

The qualification of the PK Ratios can be performed using getPKRatioMeasure. This function return a data.frame with the PK ratios within specific ranges. As a default, these ranges are within 1.5 and 2 folds. However, they can be updated using the option ratioLimits = when running the function.

# Test of getPKRatioMeasure
PKRatioMeasure <- getPKRatioMeasure(data = pkRatioData, 
                                    dataMapping = pkRatioDataMapping)
